### 背景 / 目的
アプリのランディングページとして、昆虫食への興味を引き出す紹介コンテンツと
診断・一覧への導線を実装する。#3（昆虫一覧）と #6（診断結果）が完成した後に着手する。

- 依存: #3, #6
- ラベル: `frontend`

### スコープ / 作業項目

- `features/top/pages/top_page.dart` に `TopPage` を実装する
  - アプリロゴ・キャッチコピー
  - ターゲット説明（こんな人におすすめ）
  - 昆虫食のメリット紹介
  - 診断の3つの耐性説明（👁 見た目・🤚 食べる勇気・💪 挑戦する気持ち）
  - 「診断開始」ボタン → `/diagnosis` に遷移
  - 「昆虫一覧を見る」ボタン → `/insects` に遷移
- `app.dart` の `/` ルートに `TopPage` を接続する

> **注意**: `top` 機能はページ表示のみでAPI通信がないため `repositories/` と `providers/` は不要。`pages/` のみで完結する。

### ゴール / 完了条件（Acceptance Criteria）

- [ ] `features/top/pages/top_page.dart` に `../../../docs/05_sitemap.md` 記載の主要UI要素（ターゲット説明・メリット・診断説明）が表示される
- [ ] 3つの耐性（👁 見た目への耐性 / 🤚 食べる勇気 / 💪 挑戦する気持ち）の説明が表示される
- [ ] 「診断開始」ボタンをタップすると `/diagnosis` に遷移する
- [ ] 「昆虫一覧を見る」ボタンをタップすると `/insects` に遷移する
- [ ] `flutter run` でトップ画面からすべての画面に遷移できることを手動確認できる

### テスト観点

- 手動確認（全画面遷移チェック）:
  - トップ → 診断フロー → 診断結果 → 昆虫詳細 の一連フローが通ること
  - トップ → 昆虫一覧 → 昆虫詳細 → 戻る の流れが通ること
- 検証方法: エミュレーターでの手動操作
